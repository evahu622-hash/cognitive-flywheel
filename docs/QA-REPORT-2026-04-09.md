## P0 — 专业度/第一印象问题

### 1. 浏览器标签显示 "Create Next App"

- **文件**: `src/app/layout.tsx:15-16`
- **现象**: 所有页面 `document.title` 都是 Next.js 默认模板文本
- **影响**: 多标签切换无法识别产品；竞赛评委第一印象扣分
- **修复**: 改为 `"Cognitive Flywheel 认知飞轮"` + 合适的 description

### 2. HTML lang 属性与界面语言不匹配

- **文件**: `src/app/layout.tsx:27`
- **现象**: `<html lang="en">` 但整个产品是中文界面
- **影响**: 屏幕阅读器语音错误、浏览器弹出翻译提示、SEO 语言标记错误
- **修复**: 改为 `lang="zh-CN"`

## P1 — 功能缺陷

### 3. 登录错误提示为英文

- **文件**: `src/app/auth/login/page.tsx:39`
- **现象**: 输入错误密码后显示 "Invalid login credentials"（Supabase 原始错误）
- **影响**: 中文界面里突然出现英文报错，用户困惑
- **修复**: 加一个错误信息映射，如 `"Invalid login credentials"` → `"邮箱或密码错误"`

### 4. Me 页"近7天成长曲线"只有 2 个数据点

- **文件**: `src/app/(app)/me/page.tsx:196-212`
- **现象**: 标题写"近7天成长曲线"，但图表只显示 04-02 和 04-09 两天
- **原因**: API `get_cognitive_stats` 只返回有数据的日期，前端没有补零
- **影响**: 5 天空白，图表信息量极低，给人"产品没什么在发生"的负面感受
- **修复**: 前端补齐 7 天数据，无活动的日期显示为 0

### 5. Me 页"最近成就"完全硬编码

- **文件**: `src/app/(app)/me/page.tsx:251-279`
- **现象**: 写死了 "3月15日 首次跨域连接"、"3月19日 知识条目突破 40" 等
- **影响**: 用户实际有 661 飞轮转数、658 知识条目，但成就还是"突破 40"，数据矛盾破坏信任
- **修复**: 基于真实统计动态生成成就里程碑

### 6. Evals 页面暴露给普通用户

- **文件**: `src/components/layout/app-sidebar.tsx:21` + `src/components/layout/mobile-nav.tsx:11`
- **现象**: Evals（开发者评估工具）出现在所有用户的侧栏和移动端底栏
- **控制台错误**: 2 个 503（eval 表未创建）
- **影响**: 普通用户看到 "Trace Queue"、"入口 all/feed/think" 完全不理解；移动端底栏 5 个位置浪费了 1 个
- **修复**: 从导航中移除，或加开发者模式开关/环境变量控制

## P2 — 体验问题

### 7. Feed 消化成功后缺少醒目反馈

- **文件**: `src/app/(app)/feed/page.tsx:138`
- **现象**: 消化完成后进度条 1.5 秒消失，结果静默出现在下方，无 toast/动画
- **影响**: 用户可能不确定操作是否成功，尤其是内容较长需要滚动才能看到结果时

### 8. Feed 页空状态提示与输入卡片重复

- **文件**: `src/app/(app)/feed/page.tsx:339-346`
- **现象**: 输入卡片上方已有"粘贴链接、输入想法"的引导，下方空状态区又重复一遍
- **影响**: 视觉冗余，页面显得空

### 9. Memory 页无分页，大数据量性能风险

- **文件**: `src/app/(app)/memory/page.tsx` + `src/app/api/knowledge/route.ts:57`
- **现象**: 658 条数据一次性渲染（API limit 50 但前端无分页加载更多）
- **影响**: 浏览器 snapshot 超时（测试中实际遇到），随数据增长会越来越慢

### 10. Knowledge API ilike 查询缺少输入转义

- **文件**: `src/app/api/knowledge/route.ts:61`
- **现象**: `query.or(\`title.ilike.%${search}%,...\`)` 直接拼接用户输入
- **影响**: 输入 `%` 或 `,` 等特殊字符会破坏 PostgREST 查询逻辑

### 11. Stats API 错误时静默回退到 mock 数据

- **文件**: `src/app/api/stats/route.ts:36-38`
- **现象**: 任何数据库错误都返回 `COGNITIVE_PROFILE`（mock），用户看到假数据但以为是真的
- **影响**: 数据不准确但无任何提示

## P3 — 细节优化

### 12. Think 模式卡片缺少视觉区分

- **文件**: `src/app/(app)/think/page.tsx:38-75`
- **现象**: 4 个模式全部灰色图标、白色卡片，没有颜色/图标区分
- **影响**: 用户不知道该选哪个，缺少场景引导

### 13. Sidebar 登出按钮与 Me 页登出按钮重复

- **文件**: sidebar `app-sidebar.tsx:69-75` 和 me 页 `me/page.tsx:92-95`
- **现象**: Sidebar 底部和 Me 页右上角都有登出功能，实现独立
- **影响**: sidebar 的 email 来自服务端 prop，Me 页来自客户端 fetch，可能不同步

### 14. 认知教练盲区严重程度 emoji 语义反了

- **文件**: `src/app/(app)/think/page.tsx:152-153`
- **现象**: high severity 用 🔹（蓝色），非 high 用 🔸（橙色）
- **影响**: 视觉上橙色通常代表更严重，与 severity 含义冲突

### 15. Geist 字体只加载 latin 子集

- **文件**: `src/app/layout.tsx:6-7`
- **现象**: `subsets: ["latin"]`，中文回退到系统字体
- **影响**: 中英文混排时字体风格不一致（中文用系统默认，英文用 Geist）

## 产品价值评估

### 高价值功能

1. **Feed 喂脑** — 核心价值最高。"输入任意内容 → 结构化存储 + 自动关联"解决了"看完就忘"的痛点
2. **Think 圆桌会议** — 体验惊艳。多位真实人物视角分析，用户自己做不到
3. **飞轮回流机制** — 思考产出 → 存入记忆的闭环设计简洁有效
4. **Me 认知画像** — 领域分布 + 飞轮转数让用户看到成长

### 待改进功能

1. **Memory 记忆宫殿** — 目前只是搜索列表，缺少可视化（知识图谱/时间线）来体现"宫殿"概念
2. **Think 模式选择** — 4 个模式太平，需要场景引导
3. **Evals** — 不应面向普通用户
