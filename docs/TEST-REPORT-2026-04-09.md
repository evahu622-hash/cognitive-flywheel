## 三、Feed 喂脑

### TC-09 URL 平台检测 badge
- **预期**: 输入 YouTube URL 时显示 "YouTube" badge；Twitter URL → "X/Twitter"；普通网页 → "网页"
- **实测**: ✅ 通过（YouTube URL 正确显示 "YouTube" badge，textarea label 切换为"你的想法（可选）"）

### TC-10 URL 字段 UX 陷阱（Bug）
- **预期**: 仅当输入合法 URL 时才切换 textarea label 和处理模式
- **实测**: ❌ **失败** — 输入任意非 URL 文本（如"这是一段普通文字"）时，textarea label 也会变为"你的想法（可选，会和原文一起消化）"，诱导用户误以为进入了 URL 模式
- **严重影响**: 用户误将文章内容粘贴进 URL 框时，API 会尝试将文本当作 URL 发起请求，直接报错（Evals trace 有真实失败记录：`"Failed to parse URL from Agent 架构中的 Memory 模块..."`）
- **文件**: `src/app/(app)/feed/page.tsx`，条件 `{url ? "你的想法..." : "输入想法..."}`

### TC-11 文本提交 + SSE 流式进度
- **预期**: 提交文本 → 显示处理阶段（"外脑正在理解..."、"搜索历史关联..." 等）→ 展示结果卡
- **实测**: ✅ 通过（SSE 阶段动画正常，结果卡渲染完整）

### TC-12 新功能：知识关联分析
- **预期**: 结果卡包含"知识关联分析"区块，显示关系类型 badge（支持/矛盾/扩展/不同观点）
- **实测**: ✅ 通过（实测显示 2 条"不同观点"、1 条"扩展"关联，含关联理由文本）

### TC-13 新功能：跨域闪念
- **预期**: 结果卡包含"跨域闪念"区块，amber 渐变背景，含灵感来源标注
- **实测**: ✅ 通过（实测显示竹子根系类比，来源"植物学/生命科学"，视觉效果突出）

### TC-14 成功反馈可见性
- **预期**: 消化完成后有明显成功提示（toast 或动画）
- **实测**: ⚠️ 部分通过 — 进度卡 1.5s 后消失，结果静默出现在下方；无 toast、无声音。用户如果不滚动可能不确定是否成功
