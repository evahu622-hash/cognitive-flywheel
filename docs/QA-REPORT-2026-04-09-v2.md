### 4. Me 页成长曲线仅显示有数据的日期（无补零）

- **文件**: `src/app/(app)/me/page.tsx:198-211`
- **验证**: 图表仅渲染 04-02 和 04-09，中间 5 天空白
- **根因**: `p.recentGrowth` 直接映射，未补齐 7 天。API `get_cognitive_stats` 只返回有数据的日期
- **修复**:
  ```ts
  const today = new Date();
  const last7 = Array.from({ length: 7 }, (_, i) => {
    const d = new Date(today);
    d.setDate(d.getDate() - (6 - i));
    return d.toLocaleDateString("zh-CN", { month: "2-digit", day: "2-digit" }).replace("/", "-");
  });
  const growthMap = Object.fromEntries(p.recentGrowth.map(d => [d.date, d.items]));
  const filledGrowth = last7.map(date => ({ date, items: growthMap[date] ?? 0 }));
  ```

## P2 — 体验与安全问题

### 7. Knowledge API 搜索输入未转义（安全/稳定性）

- **文件**: `src/app/api/knowledge/route.ts:72`
- **代码**: `query.or(\`title.ilike.%${search}%,summary.ilike.%${search}%\`)`
- **风险**: 输入 `%`、`,` 等特殊字符会破坏 PostgREST 查询语法，触发 500/400
- **修复**: `search.replace(/%/g, "\\%").replace(/,/g, "\\,")` 或使用参数化查询

### 8. Feed URL 输入框 UX 陷阱（新发现）

- **文件**: `src/app/(app)/feed/page.tsx:211`
- **现象**: URL 框中一旦有任何文字，textarea 的 label 立刻变为"你的想法（可选，会和原文一起消化）"；若用户误把文章内容粘贴进 URL 框，API 会尝试将该文本当 URL 请求，直接报错
- **Evals trace 证据**: 出现 `feed error: "Failed to parse URL from Agent 架构中的 Memory 模块..."` trace
- **修复**: 对 URL 框输入做基础格式校验（`url.startsWith("http")`），只有通过时才切换 label 和走 URL 处理分支

### 9. Feed 消化完成无醒目反馈（未修复）

- **文件**: `src/app/(app)/feed/page.tsx`
- **现象**: 消化完成，进度条 1.5s 消失，结果静默出现在下方，无 toast/声音/动画
- **影响**: 用户不确定是否成功，尤其是长内容需要滚动才能看到结果

### 10. Feed 空状态说明与输入卡片文案重复（未修复）

- **文件**: `src/app/(app)/feed/page.tsx:399-406`
- **现象**: 输入卡片已有引导，下方空状态区又重复说"粘贴链接、输入想法"
- **修复**: 删除空状态区的冗余文案，或换成更有趣的示例场景

## 功能价值重新评估

### 高价值（本次测试确认）

1. **Feed + 知识关联分析** — 新增关系分类（支持/矛盾/扩展/不同观点）让摘要不只是"存"，而有了"意义判断"，体验升级显著

3. **Think + 记忆层注入** — 圆桌会议自动引用了用户的历史洞察（Farzapedia、认知飞轮问题），极大增强了个人化深度
4. **飞轮回流闭环** — 思考 → 存入记忆 → Memory 页顶部即时出现，飞轮转数 +1，闭环感极强

### 值得优化

1. **Me 成长曲线** — 数据真实但视觉缺失（5 天空白），修复补零后价值会大幅提升
2. **Me 成就系统** — 672 转数是真实成就感的核心，但被"突破 40"的假数据破坏了。一旦动态化，会成为强留存钩子
