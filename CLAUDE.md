# Cognitive Flywheel 认知飞轮

## 产品定位
AI 作为人类的"外脑"，不是替代人，而是扩展人的认知能力。
核心理念：**这不是一个工具，是一个和你一起成长的外脑。**

## 核心概念：认知飞轮
```
输入 → 记忆 → 思考 → 洞察 → 回流（自强化）
  ↑                              │
  └──────────────────────────────┘
    每转一圈，外脑更懂你
```
关键设计：思考层的所有产出会压缩回流到记忆层，形成自强化循环。飞轮越转越快，外脑越用越聪明。

## 用户痛点（人脑的局限性）
- 带宽极度有限，注意力有限
- 不擅长并行任务
- 记忆力和存储不好
- 时间维度局限——只活几十年，无法借鉴几千年的人类智慧
- 知识是孤岛，难以跨领域迁移

## 时代背景（结构性变化）
- 算力不再稀缺，AI 可进行更复杂全面的逻辑推理
- 知识不再稀缺，AI 训练过程中拥有比任何个人更多知识
- 技能重要性下降，AI skills 已经很强大
- 人脑有限的带宽和算力应该用于更重要的决策

## 产品架构

### 三层架构
```
输入层：阅读/粘贴/链接/对话 → 外脑消化
            ↓
记忆层（地基）：
  ├── 知识库：读过的、学过的一切
  ├── 思考档案：观点、判断、推理过程
  └── 个人画像：领域、偏好、认知水平、盲区地图
            ↓
思考层（四大能力，全部基于记忆层驱动）：
  ├── 圆桌会议（调用知识库+思考档案）
  ├── 认知教练（调用个人画像+知识库）
  ├── 跨域连接（调用知识库跨领域检索）
  └── 历史镜鉴（调用思考档案匹配先驱经历）
            ↓
输出层：洞察 / 决策支持 / 学习路径
```

### 四大外脑思考能力

| 能力 | 说明 | 人脑做不到的原因 |
|------|------|----------------|
| 圆桌会议 | 多位专家从不同视角挑战和完善想法 | 你只有一个脑子，无法同时站在多个专家的立场 |
| 认知教练 | 发现盲区，生成个性化学习路径 | 你不知道自己不知道什么 |
| 跨域连接 | 从其他领域借鉴灵感和类比 | 人的知识是孤岛，AI的知识是网络 |
| 历史镜鉴 | 先驱们遇到过同样的问题吗？ | 你只活几十年，人类智慧积累了几千年 |

### 自强化机制（飞轮回流）
每次思考层的输出都会沉淀回记忆层：
- 圆桌会议 → 记录你采纳了哪个观点、为什么、最终决策
- 认知教练 → 更新盲区地图（已补的不再推）
- 跨域连接 → 保存有启发的跨域类比
- 历史镜鉴 → 保存你从先驱经历中获得的启示

## 页面结构（V0 Demo）

| 页面 | 路由 | 功能 |
|------|------|------|
| Landing | `/` | 品牌展示 + 导航入口 |
| Feed（喂脑） | `/feed` | 粘贴链接/文本，外脑自动消化入库 |
| Memory（记忆宫殿） | `/memory` | 知识库全局视图，搜索、筛选、关联展示 |
| Think（思考室） | `/think` | 四大能力入口，选择模式后深度思考 |
| Me（认知画像） | `/me` | 飞轮转数、知识地图、盲区地图、成长曲线 |

### 核心交互流程
1. Feed 喂脑 → 外脑自动提炼要点、打标签、存入 Memory、提示历史关联
2. Think 思考 → 从 Memory 检索上下文 → 生成多视角回应 → 用户标记有价值的观点
3. 回流 → 思考结果压缩回流 Memory → Me 页飞轮+1
4. Me 页 → 自动更新知识覆盖面、盲区变化、推荐下一步

## 技术栈

- **框架**: Next.js 15 (App Router)
- **UI**: shadcn/ui + Tailwind CSS
- **语言**: TypeScript
- **部署**: Vercel（计划中）

### AI 模型策略：多模型按需调度
| 场景 | 推荐模型 | 原因 |
|------|---------|------|
| 深度推理（圆桌、教练） | Claude / GPT-4o / DeepSeek | 推理质量高 |
| 文章摘要、标签 | Haiku / GPT-4o-mini | 快且便宜 |
| 向量嵌入 | OpenAI Embedding / 本地模型 | 知识检索用 |

通过 Vercel AI SDK 统一接口，切换模型只改一行配置，不锁死任何供应商。

### 计划中的技术组件
- 向量数据库（Supabase pgvector / Pinecone）用于知识检索
- Vercel AI SDK 统一多模型调用
- 用户认证（计划中）
